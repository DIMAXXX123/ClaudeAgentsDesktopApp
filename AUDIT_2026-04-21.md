# Ultronos Audit — 2026-04-21

## Ключевые выводы

1. **Сайт и desktop/renderer синхронизированы на 99%.** В desktop даже больше: `TitleBar`, `WindowChrome`, `uiEventBus`, `conductorFs` с `ULTRONOS_DATA_DIR`, `notify` через `window.ultronos`. Никакого «переноса с сайта» не требуется.

3. **TG listener идентичен в обеих версиях.** Работает через polling `/api/tg-feed` + `/api/listener-health` + PS WMI.
4. **Найдено 13 CRITICAL / 12 HIGH / ~15 MED багов** в 3 слоях (electron, api, frontend). 5 CRIT в Electron уже пофикшены.

## Per-agent Console — design summary

**Рекомендованный подход: гибрид.**
- Главный поток: `@anthropic-ai/claude-agent-sdk` (уже в deps) + optional `child_process` для bash
- Per-agent transcript: `userData/agents/<agentId>/transcript.ndjson` (append-only, ротация на 10k строк)
- IPC каналы: `ultronos:agent:{spawn, input, kill, restart, list, transcript, clear-transcript}` + stream event `ultronos:agent:output` + `ultronos:agent:status`
- UI: `AgentConsole.tsx` (одна панель) + `AgentConsoleGrid.tsx` (3×2 для 6 агентов) + `useAgent(sessionId)` hook
- Рендер вывода: **ansi_up** (4KB) + sanitized `<div>`, не xterm.js (heavy)
- Для web-версии: fallback disabled (только desktop).

**Phased delivery:**
- **Phase A** (IPC + registry + persistence): 2-3 дня
- **Phase B** (UI: AgentConsole + Grid + useAgent): 2-3 дня
- **Phase C** (hotkeys, export logs, search, E2E): 2 дня

**Риски:**
1. Memory leak в process streams → unsubscribe + destroy() на kill
2. Transcript bloat → ротация 10k строк
3. Race при concurrent write → write-.tmp + rename или p-queue
